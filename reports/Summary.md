# Отчет о проведенной автоматизации тестирования

## Запланированные работы

В рамках проекта планировалось выполнение следующих задач:

- тестирование UI формы, базы данных и оформление дефектов (Issues);
- настройка запуска тестируемого приложения (SUT) с подключением к БД;
- тестирование API и фиксация выявленных дефектов;
- разработка автоматизированных тестов;
- выполнение полного прогона тестов и анализ результатов;
- формирование отчетов по результатам тестирования (Allure Report);
- подготовка итоговой отчетной документации;
- настройка CI-процесса для автоматического запуска тестов.

## Фактически выполнено

В ходе реализации проекта было выполнено:

- автоматизация UI тестов формы покупки тура банковской картой(оплата и покупка в кредит);
- автоматизация API тестов сервиса;
- подключение и настройка Allure Report для формирования отчетности;
- выполнение интеграции тестов с базой данных для проверки состояния записей.

## Сработавшие риски

В процессе работы подтвердились следующие ограничения:

- отсутствие полноценной документации на приложение;
- отсутствие четких требований к валидным значениям полей формы;
- отсутствие документации по структуре базы данных и схемам таблиц;
- отсутствие формализованной спецификации API.

Данные факторы повлияли на стабильность тестирования и точность ожидаемых результатов.

## Общий итог по времени выполнения

- Планируемая трудоемкость: ~150 часов
- Фактически затрачено: ~80 часов

Сокращение трудозатрат связано с уточнением требований в процессе работы, а также с ограничением объема тестируемого
функционала из-за частичной неготовности модуля оформления кредита. В результате основное внимание было сосредоточено на
ключевых пользовательских сценариях оплаты банковской картой и базовой проверке кредитного сценария.

## Вывод

Автоматизация тестирования позволила выявить значительное количество дефектов в UI и API слоях приложения, связанных с
недостаточной валидацией данных и отсутствием единых контрактов взаимодействия между компонентами системы.
